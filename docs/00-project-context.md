# Контекст проекта

## Продуктовая задача

Система должна поддерживать создание АСОР в Cost/ESCost на основе Excel-файла АНР, приложенного в Контур.X. Прототип сфокусирован на моменте, когда созданный АСОР открывает сметчик и должен проверить результат обработки.

## Основной пользователь

Сметчик проверяет строки, созданные автоматизированной логикой. Ему нужно понимать, какие строки надежны, какие требуют внимания и почему система выбрала конкретное сопоставление.

## Границы первой версии

В прототип входит:

- индикация строк, созданных ИИ;
- предупреждения и ошибки, связанные со строками и ячейками;
- объяснение решений системы;
- навигация между списком предупреждений и строками таблицы;
- действия пользователя и комментарии в демонстрационном состоянии страницы.

В первый слой прототипа не входит:

- реальный парсинг АНР;
- реальная интеграция с сервером;
- полноценный интерфейс журнала обработки;
- полное параллельное сравнение АНР и АСОР;
- автоматическое исправление всех неоднозначных данных.

## Правила доверия

- У каждого автоматического решения должен быть видимый источник.
- Неоднозначные сопоставления нельзя скрывать.
- Ошибки нельзя отображать как успешно обработанные позиции.
- Если есть предупреждения, путь ручной проверки должен оставаться видимым.
