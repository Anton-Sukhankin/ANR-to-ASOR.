# Пользовательские пути

## Сценарий 1. Проверка строк, созданных ИИ

1. Сметчик открывает таблицу АСОР.
2. Строки, созданные ИИ, помечены в отдельной колонке индикации.
3. Сметчик открывает дровер проверки из кнопки или индикатора проблемы.
4. Сметчик фильтрует список по созданным ИИ строкам или нерешенным проблемам.
5. Сметчик выбирает позицию и переходит к соответствующей строке таблицы.

## Сценарий 2. Закрытие предупреждения

1. Сметчик нажимает на маркер предупреждения в таблице.
2. Дровер открывает карточку предупреждения.
3. Карточка объясняет исходное значение, выбранное значение в АСОР, уверенность и альтернативы.
4. Сметчик подтверждает выбор, выбирает другой вариант или оставляет комментарий.
5. Счетчики и состояние строки сразу обновляются в состоянии прототипа.

## Сценарий 3. Обработка ошибки

1. Сметчик фильтрует карточки дровера по ошибкам.
2. Сметчик открывает карточку ошибки.
3. Таблица прокручивается к проблемной строке и подсвечивает ячейку.
4. Сметчик отправляет позицию на ручной разбор или отмечает ее решенной после исправления.
5. Строка сохраняет видимый статус до тех пор, пока проблема остается открытой.

## Сценарий 4. Переход к скрытой строке

1. Строка скрыта внутри свернутой группы или работы.
2. Сметчик нажимает на карточку проблемы в дровере.
3. Родительские строки раскрываются автоматически.
4. Таблица прокручивается к целевой строке.
