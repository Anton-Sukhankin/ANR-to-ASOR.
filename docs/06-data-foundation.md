# Слой данных

## Назначение

Слой данных отделяет продуктовые данные от кода отрисовки. Благодаря этому таблица и дровер проверки могут вырасти из визуального прототипа в полноценный процесс проверки ИИ, не пряча важные сущности внутри `app.js`.

## Что изменилось

- Строки таблицы хранятся как плоский массив `asorRows` и преобразуются в дерево во время работы страницы.
- Предупреждения и ошибки связаны с исходными строками АНР и решениями ИИ по сопоставлению.
- Решения по сопоставлению содержат уверенность, порог, название алгоритма и шаги объяснения.
- Справочные альтернативы хранятся как самостоятельные кандидаты с оценками уверенности.
- Комментарии, действия пользователя и события журнала обработки вынесены в отдельные коллекции.

## Потребители данных в интерфейсе

- Тело таблицы читает `asorRows`.
- Бейджи ИИ читают `asorRow.sourceType`.
- Маркеры предупреждений и ошибок читают `issues`.
- Список позиций в дровере читает `issues`.
- Детальный разбор в дровере собирает данные по цепочке `issue -> sourceAnrRow -> matchDecision -> referenceCandidates`.
- Действия во время работы страницы добавляются в коллекции `reviewActions` и `comments` в памяти.

## Почему это важно

Следующий интерфейсный слой, «Разбор строки», требует больше данных, чем заголовок предупреждения. Он должен показывать исходную строку АНР, созданную строку АСОР, шаги рассуждения, уверенность и порог, альтернативы из справочников и финальное решение сметчика. Этот документ описывает структуру данных, которая делает такой сценарий возможным.
